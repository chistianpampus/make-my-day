# Sprint 4: Database & Backlog Management

**Focus:** Set up the Cloud Database to persistently store the user's backlog of tasks and their daily routines.

## 📋 Tasks for the Agent

- [ ] **Database Setup & Schema:** Configure the cloud database (e.g., Supabase) and create tables for `Tasks` and `Routines`.
- [ ] **Task CRUD Operations:** Implement API routes to Create, Read, Update, and Delete tasks in the backlog.
- [ ] **Routine Configuration UI:** Build a simple settings page or modal for the user to define their static routines (e.g., bedtime at 10 PM).
- [ ] **Backlog UI:** Create a view where the user can manually see all unassigned, open tasks in their backlog.
